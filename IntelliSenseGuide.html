IntelliSense Guide

Presto Xbase++ IntelliSense delivers fast, deterministic code completion based on the PSI tree and stub-indexed symbols. This guide explains how completion works, how to trigger it, and how the system is tested for reliability.

Completion Features

  • Suggestions for function and variable declarations discovered in PSI.
  • Deterministic ordering and stable ranking to avoid flickering results.
  • Insert handlers that add () for functions and position the caret.
  • Automatic case sensitivity: lowercase prefixes are matched case-insensitively.

How Completion Works

  1. CompletionContributor activates for Xbase++ files.
  2. XbCompletionService walks the PSI tree and filters by prefix.
  3. Lookup mapping enriches items with insert text and type labels.
  4. Insert strategy handles function call insertion and caret placement.
  5. Case policy switches to case-sensitive matching when uppercase letters are used.

Usage

Trigger completion with Ctrl+Space while editing .xb, .prg, or .ch files. Type a prefix to filter results.

Testing & Quality Gates

  • Completion fixtures are validated for every operations.yaml entry.
  • Completion snapshots compare expected and actual ordered results.
  • Coverage gates ensure missing or invalid fixtures are detected early.
  • Insert strategies and lookup mapping are unit-tested for all branches.
  • Prefix extraction and case policy logic are validated with unit tests.

Extending Completion

  • Add new symbol kinds in XbCompletionService and update tests.
  • Extend fixture coverage under spec/xbasepp/fixtures/completion/.
  • Keep completion output deterministic (stable ordering and explicit ranking rules).

Troubleshooting

  • Ensure the file type is recognized as Xbase++.
  • Check that fixtures include a /*caret*/ marker.
  • Use uppercase prefixes when you want strict case-sensitive matches.
  • If completion seems empty, confirm PSI is generated and symbols exist.
